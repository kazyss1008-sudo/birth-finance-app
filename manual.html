1. 環境構築・セットアップ

前提条件

  • Node.js 18 以上
  • PostgreSQL データベース（Neon, Supabase, ローカル等）
  • npm または yarn

ローカル起動手順

1
依存パッケージのインストール 
cd birth-finance-app
npm install
2
環境変数の設定

.env.example をコピーして .env を作成し、値を設定します。

cp .env.example .env
# .env の中身
DATABASE_URL="postgresql://USER:PASSWORD@HOST:5432/birth_finance?sslmode=require"
SESSION_COOKIE_NAME="birth_finance_session"
SESSION_SECRET="任意のランダム文字列（32文字以上推奨）"
3
データベースのマイグレーション 
npx prisma migrate dev --name init
4
初期データの投入（Seed）

管理者ユーザー（admin）と17種類の経費カテゴリが作成されます。

npm run db:seed
5
開発サーバーの起動 
npm run dev

ブラウザで http://localhost:3000 を開きます。

Vercel へのデプロイ時は、Vercel の環境変数設定画面で同じ変数を設定してください。

2. 初回パスワード設定・ログイン

初回パスワード設定

1

ログイン画面で admin と任意のパスワードを入力して「ログイン」をクリックします。

パスワードが未設定の場合、自動的に「初回パスワード設定」画面に遷移します。

2

8文字以上のパスワードを入力し、確認用にもう一度入力して「パスワードを設定」をクリックします。

3

設定完了後、ログイン画面に戻ります。設定したパスワードでログインしてください。

パスワードは bcrypt でハッシュ化されて保存されます。平文で保存されることはありません。

通常ログイン

ログインID（admin）とパスワードを入力して「ログイン」ボタンをクリックします。セッションは8時間有効です。

3. 公演選択

ログイン後、登録済みの公演がカード形式で一覧表示されます。

各カードには以下が表示されます：

  • ステータス（準備中 / 進行中 / 終了）
  • 公演名
  • 期間（開始日 〜 終了日）

カードをクリックすると、その公演の管理画面に遷移します。

右上の「新規公演作成」ボタンからウィザード画面に遷移できます。

4. 公演新規作成（ウィザード）

4ステップのウィザード形式で公演を作成します。全ステップ完了後に一括保存されます。

Step 1: 公演基本情報

項目説明必須
公演名公演のタイトル必須
開始日公演の開始日任意
終了日公演の終了日任意
ステージ数次のステップで行が自動生成されます任意
デフォルトノルマ単価キャスト登録時の初期値になります任意

Step 2: ステージ登録

Step1のステージ数に応じて行が自動生成されます。行の追加・削除も可能です。

Step 3: キャスト登録

キャスト名、ノルマ枚数、ノルマ単価、バック対象を設定します。

キャスト名は CSV 取込時の「取扱窓口」と完全一致させる必要があります。前後の空白は自動除去されます。

Step 4: チケットバックルール

段階式のバックルールを設定します。最大枚数を空欄にすると「上限なし」として扱われます。

5. CSV取込

公演管理画面の「CSV取込」タブからCSVファイルをアップロードします。

CSV要件

  • 文字コード: CP932（Shift-JIS）または UTF-8
  • 必須ヘッダー: 取扱窓口, 枚数, 合計額, 公演日時

処理ルール

  • 「取扱窓口」の値でキャスト名と照合します（完全一致、前後空白除去のみ）
  • 1件でもキャスト名が一致しない場合、全件ロールバックしてエラーを表示します
  • 再取込は公演単位で全削除→再登録です（差分更新ではありません）
  • 金額文字列の , と 円 は自動除去されます

エラー時の表示

不一致があった場合、以下が表示されます：

  • 不一致件数
  • 行番号と不一致だったキャスト名
  • 全件ロールバックした旨のメッセージ

8. 精算管理

「精算」タブで精算記録の登録・一覧表示・削除ができます。

画面上部にはユーザー別の精算サマリ（経費立替合計・精算済合計・残高）が表示されます。

経費と精算は直接JOINせず、それぞれをユーザー単位で先に集計してから結合しています。これにより件数の掛け算による集計誤りを防いでいます。

9. 収支確認

「収支」タブで公演全体の収支を確認できます。

サマリカード

項目計算式
総売上SUM(sales.sales_amount)
総経費SUM(expenses.amount)
ギャラSUM(キャストごとのバック精算額)
最終収支売上 - 経費 - ギャラ

キャスト別バック試算

各キャストの販売枚数・売上・バック総額・ノルマ控除・精算額が表示されます。

「内訳」リンクから各キャストの詳細精算画面に遷移できます。

チケットバックの計算結果はDBに保存されません。画面表示のたびに都度再計算されます。

10. キャスト精算内訳

キャストごとの精算内訳を表示します（画面表示のみ、PDF出力・印刷機能は非搭載）。

表示内容

  • 販売枚数 / 売上金額 / ノルマ控除額 / 最終精算額
  • 段階式バック内訳テーブル（対象範囲・該当枚数・単価・小計）
  • 計算式の表示
  • ステージ別販売実績テーブル

バック計算ロジック

for each rule:
  applicable = max(min(N, max) - min + 1, 0)
  total += applicable × unit_price

精算額 = バック総額 - (ノルマ枚数 × ノルマ単価)

12. トラブルシューティング

「パスワードが未設定です」と表示される

初回ログイン時に表示されます。パスワード設定画面で8文字以上のパスワードを設定してください。

CSV取込で「キャスト不一致」エラー

CSVの「取扱窓口」列の値と、公演に登録されたキャスト名が完全一致しません。以下を確認してください：

  • キャスト名のスペル（全角/半角を含む）
  • 前後の空白（自動除去されますが、名前中のスペースは区別されます）
  • キャストが公演に登録済みであること

ビルドエラー「DATABASE_URL not found」

.env ファイルに DATABASE_URL が設定されていることを確認してください。Vercel の場合は環境変数設定で追加してください。
